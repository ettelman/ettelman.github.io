Tailwind CSS

Information

Tailwind CSS är ett stort ramverk som har mängder av funktioner och är väldigt kraftfullt. Precis som foundation så finns det många olika sätt att inkludera det i ett projekt men på grund av storleken så rekommenderas det inte att inkludera hela CSS filerna eftersom de är enormt stora och påverkar laddningstiderna för webbplatsen negativt. För denna uppgift har jag inkluderat Tailwind med hjälp av CLI och på så sätt endast inkluderat de klasser som faktiskt används av webbplatsen. Tailwind resettar stora delar av den normala CSS-stylingen som finns från början så att utvecklaren får möjlighet att styla varje element från början. Det finns som sagt enormt många klasser och tailwind ersätter i princip egenskriven CSS helt. Detta gör att Tailwind passar bra till både stora och små projekt och när man väl lärt sig att använda det är det väldigt smidigt.

Fördelar och nackdelar

Som nämts ovan är Tailwind väldigt anpassningsbart och fungerar för i princip vilken webbplats som helst. Det är smidigt att lära sig och klassnamnen är lätta att förstå och lista ut. Det är snabbt att ladda om man kompilerar sin egen CSS fil och det finns väldigt mycket funktionalitet. Tailwind tar dock ganska mycket tid att lära sig och eftersom många element "resettas" så måste man spendera tid på att styla i princip varje element vilket tar extra tid om man ska göra något enkelt. Sen är det svårare att använda Tailwind för någon som inte har jobbat med terminaler och liknande eftersom det inte rekommenderas att inkludera hela CSS filen eller att använda sig av CDN-länkar. Överlag så känns Tailwind CSS som något jag vill lära mig och det verkar väldigt kraftfullt och smidigt när man kan det!
