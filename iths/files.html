Webbtestning - Intressanta Filer

1. Konfigurationsfiler

Konfigurationsfiler kan innehålla känslig information som databasuppgifter eller serverinställningar:

  • phpinfo.php - Ger en detaljerad översikt över PHP-konfigurationen.
  • config.php, .env - Vanliga filer med databasanvändarnamn eller API-nycklar.
  • web.config - Konfigurationsfil för IIS-servrar.
  • application.yml eller .json - Vanligt i Java- eller Spring Boot-applikationer.

2. Systemfiler

Dessa filer kan avslöja serverinställningar eller felsökningsinformation:

  • robots.txt - Kan avslöja dolda sökvägar eller admin-sidor.
  • sitemap.xml - Visar en lista över sidor på webbplatsen.
  • .htaccess - Innehåller inställningar för Apache-servern.
  • .git/ - Om denna är åtkomlig kan den avslöja hela källkoden.

3. Backup- och testfiler

Utvecklare lämnar ibland kvar filer som kan innehålla känslig eller föråldrad information:

  • backup.zip, db-backup.sql, old/ - Backupfiler med känslig data.
  • test.php, test.html - Testfiler som används under utveckling.
  • index.html.bak, index.php~ - Backup- eller temporära filer.

4. Loggfiler

Loggfiler avslöjar ofta serveraktivitet eller användarinteraktioner:

  • error.log, access.log - Vanliga på Apache och Nginx.
  • debug.log - Kan innehålla detaljer för felsökning.

5. Filuppladdningar

Om applikationen tillåter filuppladdningar, kontrollera följande:

  • Om uppladdade filer är åtkomliga via en URL (t.ex. uploads/).
  • Om uppladdade filer exekveras utan korrekt kontroll.

6. Andra intressanta filer

  • README, CHANGELOG, VERSION - Metadata om applikationen.
  • install.php, setup.php - Installationsskript som inte har tagits bort.
  • crossdomain.xml, clientaccesspolicy.xml - Policys för Flash eller Silverlight.

Rekommenderade verktyg

För att identifiera dessa filer, använd följande verktyg:

  • Burp Suite: Skanna och analysera HTTP-förfrågningar och svar.
  • DirBuster eller gobuster: Bruteforce för att hitta dolda kataloger och filer.
  • Wayback Machine: Kontrollera äldre versioner av webbplatsen för exponerade filer.
  • git-dumper Analysera filer i .git katalogen
